## 🧪 ผลการทดสอบรายกรณี (Test Cases Details)

### 1. ทดสอบการรันเว็บแอปด้วย Streamlit
* **สิ่งที่ทดสอบ:** เรียกใช้งานคำสั่ง `streamlit run web_app.py`
* **ผลลัพธ์ที่คาดหวัง:**
  - ตัวรันของระบบทำงานสำเร็จ เปิดหน้าบราวเซอร์ไปที่พาธ `http://localhost:8501` อัตโนมัติ
  - หน้าจอแสดงส่วนป้อนฟอร์ม ซ้อนช่องป้อนสถานะ และส่วนแสดงแดชบอร์ดประวัติคิวงานโดยไม่มีสคริปต์ Error
* **ผลการทดสอบจริง:** **ผ่านการทดสอบ (PASSED)**

### 2. ทดสอบการเขียนเพิ่มแถวใหม่จากหน้าฟอร์ม (Form Submission)
* **สิ่งที่ทดสอบ:** กรอกหัวข้อ Topic = "แนะนำเครื่องมือจัดการโปรเจกต์สำหรับนักพัฒนา" และ Keyword = "เครื่องมือจัดการโปรเจกต์" แล้วกดส่ง
* **ผลลัพธ์ที่คาดหวัง:**
  - ฟอร์มล้างช่องข้อมูลให้ว่าง (clear_on_submit)
  - หน้าจอแสดงข้อความยืนยันสีเขียวบอกตำแหน่งแถวที่เซฟจริง
  - ในตาราง Google Sheet จะมีแถวใหม่ถูกแทรกท้ายสุด โดยมี Status = `Waiting` และวันที่ `Created At` ปรากฏ
* **ผลการทดสอบจริง:** **ผ่านการทดสอบ (PASSED)**

### 3. ทดสอบการเช็คคิวและสถานะงานเรียลไทม์ (Status Tracker)
* **สิ่งที่ทดสอบ:** เมื่อแถวถูกจองแล้ว ให้สั่งรัน Engine หลักหลังบ้าน (`python main.py`)
* **ผลลัพธ์ที่คาดหวัง:**
  - ตัวประมวลผล Engine ดึงหัวข้อไปแต่งและโพสต์เป็นร่างใน Blogger สำเร็จ
  - เมื่อผู้ใช้งานกลับมากดปุ่ม "🔄 โหลดรีเฟรชสถานะแถวล่าสุด" บนหน้าเว็บเพจ:
    - สถานะเปลี่ยนเป็น `Drafted`
    - แสดงกล่องข้อความสีเขียวแจ้งเตือนทำสำเร็จ
    - โชว์รายละเอียดข้อมูล SEO Title, Meta Description และมีลิงก์สีฟ้าสำหรับเปิดหน้าเอกสารบทความแบบร่างบน Blogger โชว์ขึ้นมาแทนที่ข้อความรอประมวลผล
* **ผลการทดสอบจริง:** **ผ่านการทดสอบ (PASSED)**

### 4. ทดสอบความถูกต้องของแดชบอร์ดภาพรวม (Dashboard Grid)
* **สิ่งที่ทดสอบ:** ตรวจเช็คข้อมูลตารางประวัติผลงานสรุปด้านล่างสุดของเว็บแอป
* **ผลลัพธ์ที่คาดหวัง:**
  - ตารางสรุปต้องโชว์รายการบทความครบถ้วนตามชีต โดยเอาคิวใหม่ที่สุดขึ้นก่อน (Reversed Sort)
  - คอลัมน์ Blogger URL ต้องแสดงเป็นลิงก์ที่สามารถคลิกเปิดหน้า Blogger ปกนั้นได้จริง
* **ผลการทดสอบจริง:** **ผ่านการทดสอบ (PASSED)**
