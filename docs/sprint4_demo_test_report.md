## 🧪 ผลการทดสอบรายกรณี (Test Cases Details)

### 1. ทดสอบเปิด Demo Mode ผ่าน URL Parameter (?demo=true)
* **สิ่งที่ทดสอบ:** เรียกใช้งานบราวเซอร์ที่พิกัด `http://localhost:8501/?demo=true`
* **ผลลัพธ์ที่คาดหวัง:**
  - หน้าจอถูกสลับเป็นโหมดทดลองใช้ของลูกค้า (Demo Mode) ทันที
  - ซ่อนประวัติตาราง สถิติแถวล่าสุด แผงควบคุมระบบ และตารางคิวงานหลังบ้านทั้งหมด
* **ผลการทดสอบจริง:** **ผ่านการทดสอบ (PASSED)**

### 2. ทดสอบความเสถียรของระบบประมวลผลทันใจ (Real-time Generator)
* **สิ่งที่ทดสอบ:** กรอกหัวข้อและข้อมูลจำลองใน Demo Mode จากนั้นกดปุ่ม "สร้าง Content Pack ทันที"
* **ผลลัพธ์ที่คาดหวัง:**
  - สัญญานกล่องแสดงสัญญาน (Status Box) แสดงขั้นตอนทีละขั้น (เขียนชีต -> เจน Gemini -> ยิง Blogger -> เซฟชีต)
  - หน้าจอค้างประมวลผล และเปิด 5 แท็บโชว์ผลงานพร้อมลิงก์ขึ้นมาทันทีเมื่อเจนเสร็จ
  - ย้อนกลับไปดูในแผ่นชีต: ข้อมูลคิวถูกลงทะเบียนเป็น `Drafted` จริง
  - ย้อนกลับไปดูใน Blogger: มี Draft ของหน้าบทความใหม่โผล่ขึ้นมาตามปกติ
* **ผลการทดสอบจริง:** **ผ่านการทดสอบ (PASSED)**

### 3. ทดสอบการคงอยู่ของระบบ Admin ปกติ (Standard Mode)
* **สิ่งที่ทดสอบ:** เปิดลิงก์ปกติ `http://localhost:8501`
* **ผลลัพธ์ที่คาดหวัง:**
  - หน้าต่าง Admin Portal และตารางสรุปแดชบอร์ดหลักด้านล่างยังคงทำงานสมบูรณ์ ไม่ถูกผลกระทบของ Demo Mode รบกวน
* **ผลการทดสอบจริง:** **ผ่านการทดสอบ (PASSED)**
