# Sprint 4 Demo Mode Plan - Client Trial

แผนการเตรียมระบบ Demo Mode (Client Trial) สำหรับให้ลูกค้านำไปทดลองใช้และปิดการขาย

## เป้าหมายของโครงการ (Goal)
สร้างหน้าตาการใช้ระบบที่เรียบง่าย ปิดซ่อนฟังก์ชันสำหรับนักพัฒนาและผู้ดูแลระบบทั้งหมด (ซ่อนประวัติตารางและแท็กสรุปเบื้องหลัง) และดึงดูดผู้ใช้งานด้วยระบบประมวลผลทันใจ (Synchronous Execution) ทันทีที่กดปุ่ม เพื่อให้ได้สัมผัสความพรีเมียมของระบบภายในเวลา 3 นาที

---

## สถาปัตยกรรมระบบและส่วนเชื่อมต่อ (Technical Architecture)

1. **การตรวจสอบสิทธิ์เข้าถึงพอร์ทัลลูกค้า:**
   - ใช้โมดูลตรวจสอบ Query Parameter จาก Streamlit: `is_demo = st.query_params.get("demo", "false").lower() == "true"`
   - เมื่อเชื่อมต่อผ่าน `?demo=true` หน้าร้านจะถอดระบบควบคุมเบื้องหลังออกและแสดงเฉพาะหัวข้ออินพุต
2. **ระบบการเรียกประมวลผลทันที (Synchronous Execution Flow):**
   - ต่างจากโหมดทั่วไปที่ข้อมูลต้องเข้าคิวเพื่อรอสคริปต์ Engine ในโหมดเดโมเมื่อคลิกปุ่มจะเรียกลำดับการประมวลผลทันทีผ่านทางหน้าเว็บ Streamlit:
     - 1. ลงทะเบียนเพิ่มแถวในชีต (`add_new_row`)
     - 2. ล็อกสถานะ `Processing`
     - 3. ส่งข้อมูลให้ Gemini API เจน JSON แพ็คคอนเทนต์ (`generate_blogger_article`)
     - 4. เชื่อมส่งบทความขึ้น Blogger REST API คืนลิงก์แบบร่าง (`create_draft_post`)
     - 5. บันทึกผลลงชีตชีตความคืบหน้าสำเร็จ (`update_row_success`)
3. **การคัดลอกผลงานสะดวก (Copy-Friendly Interface):**
   - รวมการตลาด 5 แท็บ: SEO Article, Facebook Post, TikTok Script, YouTube Shorts, Image Prompt
   - นำคุณสมบัติ `st.code` มาจัดแสดงผลลัพธ์เพื่ออำนวยความสะดวกในการกดปุ่มคัดลอก (Copy)

---

## ความเข้ากันได้ย้อนหลัง (Backward Compatibility)
- ยังคงบันทึกข้อมูลและดึง Blogger URL จากฐานข้อมูลชีตเดียวกันกับประวัติหลัก
- ไม่กระทบกลไกการวนลูปประมวลผลแบบเดิมบน `main.py`
