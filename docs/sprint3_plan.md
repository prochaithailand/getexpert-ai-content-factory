# Sprint 3 Plan - Web Form MVP

แผนการดำเนินงานของ Sprint 3 สำหรับโครงการ **GetExpert AI Content Factory**

## เป้าหมายของ Sprint (Sprint Goal)
พัฒนาหน้าต่างติดต่อผู้ใช้งานแบบหน้าเว็บอย่างง่าย (Web Form MVP) โดยนำเฟรมเวิร์ก **Streamlit** มาขับเคลื่อน เพื่อเพิ่มความสะดวกในการคีย์ส่งรายการหัวข้อและคำสำคัญ และสามารถติดตามความคืบหน้าการเขียนบทความของระบบได้โดยตรงผ่านหน้าเบราว์เซอร์

---

## สถาปัตยกรรมระบบ (System Architecture)
เพื่อรักษาระดับการประมวลผลและการขยายตัวในอนาคต สถาปัตยกรรมจะถูกแยกบทบาทออกเป็น 2 ส่วนหลัก:

1. **Content Processing Engine (main.py):**
   - ยังคงเป็นตัวประมวลผลหลังบ้าน (Backend Job Engine) ทำหน้าที่เชื่อมตารางชีต อ่านคิว รอส่ง Gemini เจน และส่ง Draft ขึ้น Blogger รวมถึงมีระบบ Retry เหมือนกับ Sprint 2 ทุกประการ
2. **User Interface / Portal (web_app.py):**
   - ทำหน้าที่เป็นหน้าตาคอยให้บริการรับเรื่อง (Frontend Portal) รับข้อมูล Topic/Keyword จากผู้ใช้ เขียนบันทึกเปิดแถวใหม่ลง Google Sheet ในสถานะ `Waiting` และดึงข้อมูลประวัติทั้งหมดมาจัดแสดงในแดชบอร์ด

---

## รายละเอียดหน้าจอ Streamlit (UI Design Elements)
* **Form Submission (ช่องกรอกข้อมูล):**
  - ช่องรับข้อมูลชื่อหัวข้อ (Topic Input)
  - ช่องรับข้อมูลคำสำคัญหลัก (Keyword Input)
  - ปุ่มกดยืนยัน (Add to Queue Button) เพื่อเพิ่มรายการใหม่เข้าไปยังชีต
* **Status Tracker (ตัวดึงดูดสถานะล่าสุด):**
  - แสดงสถานะล่าสุดของแถวงานที่ป้อนเข้ามาใหม่ในเซสชันนั้นๆ (Waiting, Processing, Drafted, Failed)
  - ปุ่มกดดึงค่าใหม่ด้วยมือ (Refresh Status Button)
  - แสดงรายละเอียด SEO Title, SEO Meta Description และ ลิงก์ตรงสำหรับคลิกเปิดหน้าบทความร่างบน Blogger ทันทีเมื่ออัปโหลดเรียบร้อย
* **Dashboard Table (ตารางแสดงประวัติรวม):**
  - แสดงตารางข้อมูลคิวงานทั้งหมดในแผ่นชีตเรียงกลับหัว (แถวใหม่ขึ้นต้นสุด) เพื่อให้อ่านมอนิเตอร์สถานะประวัติงานทั้งหมดได้ง่าย
