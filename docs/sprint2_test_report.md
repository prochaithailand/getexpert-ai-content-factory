## 🧪 ผลการทดสอบรายกรณี (Test Cases Details)

### 1. ทดสอบการรันแบบปกติสำเร็จ (Normal Execution)
* **สิ่งทื่ทดสอบ:** ใส่แถวทดสอบ 4 แถวที่มี `Status = Waiting` กรอกหัวข้อและคีย์เวิร์ดต่างกัน
* **ผลลัพธ์ที่คาดหวัง:**
  - สคริปต์กรองและเลือกเฉพาะแถว `Waiting` มารัน
  - มีการล็อกคิวขึ้นสถานะ `Processing` ในระหว่างทำงาน
  - Gemini API ส่งค่ากลับครบถ้วนในรูปแบบ Structured JSON
  - โพสต์ถูกสร้างจริงเป็นแบบร่าง (Draft) ใน Blogger
  - ชีตถูกอัปเดตกลับเป็นสถานะ `Drafted`
  - คอลัมน์ SEO E ถึง R ถูกเขียนครบถ้วน (SEO Title, Slug, คีย์เวิร์ดรอง, Prompt รูปปก, สไตล์ภาพ, เวลาประมวลผล ฯลฯ)
* **ผลการทดสอบจริง:** **ผ่านการทดสอบ (PASSED)**

### 2. ทดสอบระบบ Retry เมื่ออินเทอร์เน็ตหลุด / หรือ API ล้มเหลวชั่วคราว
* **สิ่งที่ทดสอบ:** จำลองเหตุการณ์ระบบขัดข้องชั่วคราวระหว่างเขียนบทความ
* **ผลลัพธ์ที่คาดหวัง:**
  - ระบบจะพยายามส่งคำขอใหม่สูงสุด 3 ครั้ง
  - ระยะเวลาหยุดรอเป็นไปตามกำหนดคือ 2, 5 และ 10 วินาที
  - หากกลับมาเชื่อมต่อได้ในรอบ Retry โครงการจะดำเนินงานต่อได้สำเร็จและบันทึกเป็น `Drafted`
* **ผลการทดสอบจริง:** **ผ่านการทดสอบ (PASSED)**

### 3. ทดสอบการดักจับข้อผิดพลาดรายแถว (Fault Tolerance)
* **สิ่งที่ทดสอบ:** ใส่แถวข้อมูลจำลองแถวแรกให้เกิด Error แน่ๆ (เช่น ป้อนค่าทดสอบที่ระบบตรวจสอบไม่ผ่าน หรือเจตนาป้อนคีย์ API ปลอมเฉพาะบางฟังก์ชัน) ส่วนแถวอื่นๆ ป้อนข้อมูลตามปกติ
* **ผลลัพธ์ที่คาดหวัง:**
  - แถวแรกที่เกิดปัญหา เมื่อพยายาม Retry ครบ 3 รอบแล้ว จะต้องเปลี่ยนสถานะใน Sheet เป็น `Failed` และอัปเดตช่อง `Last Error` บอกสาเหตุ และช่อง `Retry Count` บันทึกเลข 3
  - **ระบบต้องไม่หยุดการทำงานทั้ง batch** แต่จะข้ามไปทำแถวที่สองและแถวต่อๆ ไปได้จนสำเร็จสมบูรณ์
* **ผลการทดสอบจริง:** **ผ่านการทดสอบ (PASSED)**

## 🔍 ตารางยืนยัน DoD ผลลัพธ์ใน Google Sheets

ระบบยืนยันการบันทึกข้อมูลฟิลด์คอลัมน์ใหม่ครบถ้วนหลังจากรัน:
* **SEO Title:** บันทึกถูกต้อง ความยาวไม่เกิน 60 อักษร
* **Meta Description:** สรุปย่อขนาด 120-150 อักษร
* **Slug Suggestion:** คำภาษาอังกฤษแนะนำคั่นด้วยขีดกลาง
* **Related Keywords:** แสดงผลรวมคำสำคัญสำรอง
* **Featured Image Prompt:** ปรากฏข้อมูล Prompt ภาษาอังกฤษสำหรับเจนปกรูปภาพ
* **Retry Count:** อัปเดตระบุรอบการลองทำงานใหม่ (0 กรณีปกติสำเร็จ, 3 กรณีล้มเหลว)
* **Last Error:** บันทึกแสดงสาเหตุปัญหาเมื่อแถวพัง
* **Processed At:** บันทึกเวลาที่ประมวลผลเสร็จสิ้น
